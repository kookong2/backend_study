1. 데이터베이스 기본 개념

	키
	- 슈퍼키 : 투플을 유일하게 식별할 수 있는 하나의 속성
	- 후보키 : 투플을 유일하게 식별할 수 있는 하나의 속성 혹은 속성의 집합으로, 속성의 최소 집합이다.
	- 기본키 : 여러 후보키 중 하나를 선정하여 대표로 삼는 키를 말한다.
	- 대리키 : 일련번호 같은 가상의 속성을 만들어 기본키로 삼는 것을 말한다.
	- 외래키 : 다른 릴레이션의 기본키를 참조하는 속성을 말한다.

자료형

	숫자
		정수
			BIGINT(64bit)
			INT(INTEGER - 32bit)
			MEDIUMINT
			SHORTINT
			TINYINT	

		실수
			FLOAT
			DOUBLE

	문자
		한 줄 문자
			CHAR(문자 길이)
				ex) CHAR(10) : 고정 사이즈의 크기 
					-> 10자 미만의 문자를 입력해도 10의 크기를 유지

			VARCHAR(문자 길이)
				ex) VARCHAR(10) : 최대사이즈가 10자
					-> 1자를 입력하면 1의 크기를 가짐
		
		여러 줄 문자
			TEXT
			MEDIUMTEXT
			LONGTEXT

	날짜
		DATE : 날짜
		TIME : 시간
		DATETIME : 날짜 + 시간 (제일 많이 씀)
		TIMESTAMP : EpochTIme

	기타
		ENUM : 열거형
		SET : 집합

INDEXES - PRIMARY KEY (UNIQUE + NOT NULL)
(목차 - 조회)

NULL : 데이터가 없는 경우 null

제약조건
	- NOT NULL : NULL을 미허용, 데이터를 추가하지 않으면 오류 
	
	- UNIQUE : 중복이 안되는 제약조건
	
	- DEFAULT 기본값 : NULL일 때 기본값으로 대체

	- CHECK(조건식)
	
	- FOREIGN KEY (컬럼명) REFERENCES 테이블명(기본키)
		- ON DELETE CASCADE | SET NULL

	- 외래키 제약조건 : 기본값 - 연관된 데이터 삭제 불가
		- ON DELETE CASCADE : 부모 데이터가 삭제되면 자식 데이터도 삭제
		

SQL(Structured Query Language)
1) 데이터 정의어(DDL - Data Definition Language)
	- 데이터를 저장할 수 있는 틀을 정의하는 언어
	- 데이터베이스
	- 테이블

	- CREATE 문 : 데이터베이스, 테이블 생성
		- 데이터 베이스 생성
			ex) CREATE DATABASE 데이터베이스명;

		- 데이터베이스 목록 조회
			ex) SHOW DATABASES; 
		
		- 데이터베이스 선택
			ex) USE 데이터베이스명;

		
		- 테이블 생성
			ex) CREATE TABLE 테이블명(
				컬럼명 자료형 제약조건,
				...
				...
				...
			);

	- ALTER 문 : 테이블의 정의를 변경
	- DROP 문 : 데이터베이스, 테이블을 삭제
		- 데이터 베이스 삭제
			ex) DROP DATABASE 데이터베이스명;
		- 테이블 삭제
			ex) DROP TABLE 테이블명;
	
	테이블 보기
		ex) DESCRIBE 테이블 명; (= DESC 테이블명;)

2) 데이터 조작어(DML - Data Manipulation Language)
	- 데이터를 다루는 언어(추가, 삭제, 변경, 조회)
	
	- INSERT 문
	- DELETE 문
	- UPDATE 문
	- SELECT 문

3) 데이터 제어어(DCL - Data Control Language)
	- 사용자 권한, 취소

	- GRANT
	- REVOKE
