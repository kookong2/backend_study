2023.03.21

- 검색기능 

 초기 검색 기능 구현) 위치 기반으로 하는 검색, 카테고리별 스터디 검색, 통합 스터디 검색
-> 문제점 : 많은 필터를 사용해야 하고, 초기에 계획했던 '위치기반으로 하는 스터디 모집'에 대한 주제가 불명확해짐. 
검색 기능에 초점을 두었을 때, 1) 카테고리 & 스터디명 검색, 2) 위치 기반 & 스터디명 & 카테고리 검색 , 2가지 방안 중 첫 번째 방안이 깔끔한 검색 기능을 할 수 있음. 

따라서, '위치 기반'은 스터디 개설 폼에 스터디가 어디서 이루어지는지 정도로만 사용할 수 있게끔 ( 위치기반 API의 사용 빈도를 낮춤) 수정함. 

- 변경 후 기능 구현) 
	 - 카테고리 & 스터디명 검색 : OPTION( 통합검색, 카테고리(넓은 범주)검색 )형태로 검색하는 기능으로 변경.
	-  지도 API : 스터디 개설& 활동 장소 노출

- 관리자 페이지 기능

관리자 페이지에서는 스터디를 관리할 수 있는 '스터디 관리' 페이지와 '커뮤니티 관리' 페이지를 구현. 
'스터디 관리' 페이지 : 스터디 관리 목록 중 '열'에 들어갈 변수설정 완료. 
또한, 이 페이지에서는 각 스터디에 대한, '스터디 소개 페이지', '스터디 회원 페이지'를 확인할 수 있음.
 
'커뮤니티 관리' 페이지 :  앞서 배운 (BOARD) 게시판 관리 페이지와 동일하게 함. 
-> 세부 기능 미설정. 

 관리자 페이지 내에 기능에 대한 회의를 진행하면서, 관리자 페이지 화면 설계에 대한 교차점 발생
 조원들 간의 생각하고 있는 관리자 페이지가 상이하였고, 뉴마커스 관리자 페이지를 참고하여 회의를 진행하다 보니, UI 설계에 대한 문제와 페이지 세부 관리 목록에 대해 명확한 설계가 이루어지고 있지 않음을 발견 

->  문제점 : 주제에 대한 기능 구현은 완료 하였으나, 화면설계에 대한 구상이 이루어지지 않아 기능 구현을 토대로한 배치에 대해 어려움을 느낌. 따라서, 기능 구현을 어느정도 완료 하였다고 판단하여 화면설계를 함께 하기로 결정. 
또한, 관리자 페이지도 기능뿐 아니라 화면설계도 해야함을 놓치고 있었음. 

-> 해결방안 : URL을 기준으로 페이지에 대한 화면 설계를 조원들간 분배하여  03.22-03.23 (2일) 간 설계를 해오기로 함. 단, 메인페이지에서 이동하는 URL이 대부분임을 판단하여 메인페이지를 먼저 구상하고, 다른 페이지들을 분배하기로 함. 
또한, 의견 충돌이 일어났던 관리자 페이지에 대해서 '스터디목록 페이지'는 기능구현을 완료하였기에, 화면 설계를 함께 진행하기로 함.
